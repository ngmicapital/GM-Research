Claude Ecosystem
Claude Code v2.1.89 adds defer-permission decisions to PreToolUse hooks — headless sessions can now pause, wait for human review, and resume, closing the supervised-automation loop.
[Update]
⚡ Claude Code v2.1.89 — Defer Permission + Headless Resume

Released April 1. The headline change is defer as a permission decision on PreToolUse hooks. Previously, hooks could only approve or block tool calls. Now a hook can defer — pausing the headless session at that tool call and letting you resume it manually with claude -p --resume <session-id>.

Other changes in v2.1.89:

• MCP_CONNECTION_NONBLOCKING=true — in headless (-p) mode, skip waiting for MCP connection rather than timing out the whole session
• MCP connections bounded at 5s — no longer blocked by a single slow MCP server
• PermissionDenied hook fires after auto-mode classifier denials — allows custom handling when auto-mode blocks a tool
• Named subagents in @ mention typeahead — subagent addressing surfaced in the UI

This builds on v2.1.88's PermissionDenied hook (covered Issue #6). Together they form a complete supervised-automation primitive set.

⚡ Implication: This is the week to build the pattern: add defer decisions to hooks on any PreToolUse event for file writes, API calls, or deployments you want human sign-off on. dc's existing Claude Code setup gains a genuine safety checkpoint for sensitive automated tasks.
[Update]
Claude Desktop — .mcpb Extensions + OAuth Breaking Change

Claude Desktop now supports .mcpb packaged MCP servers — double-click to install, no JSON config editing required. The MCP ecosystem has 400+ community servers; the top 50 average 12,000+ monthly installs. This is the moment MCP crosses from developer primitive to user-facing app store.

Breaking change for MCP server operators: The OAuth metadata endpoint moved from claude.ai/oauth/mcp-oauth-client-metadata to claude.ai/api/oauth/mcp-oauth-client-metadata. If you maintain any MCP servers with hardcoded OAuth paths, update them now.

⚡ Implication: Check any custom MCP configurations for the OAuth endpoint. The .mcpb format means finding and installing new MCP servers just became frictionless — browse mcp.so for servers relevant to trading research and Notion workflows.

Workflow & Tools
The Claude Code defer-permission pattern is this week's highest-leverage workflow primitive — build it today for any automation touching live files, APIs, or deployments.
[Workflow] [Update]
⚡ Claude Code Defer Pattern — Build It This Week

The defer permission decision in v2.1.89 PreToolUse hooks is the missing primitive for supervised automation. Here's the pattern:

1. Add a PreToolUse hook that returns defer for any tool you want human sign-off on (e.g., file writes to production paths, API POST calls, destructive operations).
2. Run the session headless with claude -p "your task" — it will pause at the deferred tool call and print the session ID.
3. Review the pending action, then resume with claude -p --resume <session-id> to approve, or let it expire to cancel.

Combined with the v2.1.88 PermissionDenied hook, you now have full observability and control over what Claude does autonomously. Set MCP_CONNECTION_NONBLOCKING=true for headless runs to avoid MCP timeout failures.

⚡ Try today: Add a defer hook to any Claude Code briefing automation that writes files, sends emails, or calls external APIs. This is the right safety architecture for autonomous workflows.
