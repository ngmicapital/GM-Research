Claude Ecosystem
Anthropic accidentally shipped Claude Code's entire 513k-line TypeScript source inside a public npm package — the DMCA chaos that followed confirmed the architecture while open-source ports remain online.
[Update]
The Claude Code Source Leak — What Actually Happened

On March 31, 2026, Anthropic accidentally included a 59.8 MB JavaScript source map (.map) file in the public npm release of Claude Code v2.1.88. The map file contained approximately 513,000 lines of unobfuscated TypeScript across 1,906 files — the complete internal codebase. It was mirrored and forked tens of thousands of times within hours.

Technical cause: The Bun runtime used to build Claude Code generates full source maps by default. The *.map pattern was not excluded in .npmignore or the files field of package.json. The map file referenced a complete ZIP of the original TypeScript sources on Anthropic's own Cloudflare R2 bucket — publicly accessible.

The DMCA chaos: Anthropic issued a sweeping DMCA notice that GitHub executed against approximately 8,100 repositories — far beyond the intended scope. Anthropic's head of Claude Code, Boris Cherny, publicly stated the mass takedown was unintentional and retracted it, limiting the notice to one primary repository and 96 direct forks. Open-source rewrites targeting GPT, Llama, and other backends remain online — they're reimplementations, not copies, and are DMCA-immune.

What the leak confirmed: The technical content revealed the agent runtime structure, the hooks architecture (validating everything in Issues #6–7), and the KAIROS (always-on ambient Claude) and BUDDY (ambient AI companion) product features first surfaced in the Mythos leak. No customer data or credentials were exposed.

⚡ Implication: No security action required for dc — no credentials leaked. The architectural exposure is interesting (the hooks system is more sophisticated than publicly documented), but the main takeaway is practical: the open-source ports are real and advancing. The most capable spin-offs will arrive in 2–4 weeks. Watch the community build a GPT-5.4-backed Claude Code equivalent.

Research Radar
Shanghai AI Lab's PRCO framework disentangles visual perception from symbolic reasoning in multimodal models — solving a problem that's been quietly limiting every vision-language model since GPT-4V.
[Research]
PRCO — Perception-Reasoning Coevolution for Multimodal AI (Shanghai AI Lab)

PRCO (Perception-Reasoning Coevolution) is a dual-role reinforcement learning framework from Shanghai AI Lab that addresses a specific failure mode in Multimodal Large Language Models (MLLMs): the entanglement of visual perception and symbolic reasoning in a single joint training objective.

The core insight: current MLLMs mix "what do I see" and "what does it mean" into a single training signal, causing each to interfere with the other. When the model makes a reasoning error, it's ambiguous whether the perception was wrong (bad visual encoding) or the reasoning was wrong (correct perception, bad inference). PRCO separates perception and reasoning into distinct roles, trains them independently under RL, then recombines — allowing each to improve without degrading the other.

Early benchmark results show improvements on complex visual reasoning tasks. No code release confirmed at time of writing. [Results from single arXiv source — not yet independently replicated]

Why it matters in 6 months: If PRCO or its descendants ship in production multimodal models, the failure mode where Claude/GPT-5.4 confidently misreads a chart or image before reasoning incorrectly about it becomes less common. For anyone using AI to analyse market charts, financial statements, or visual data, this is directly relevant capability research.

⚡ Implication: Monitor for code release and community replication — if the framework holds up, it will surface in Qwen3.5-Omni and Gemini 3.1 Pro fine-tunes within 3–6 months. For dc's potential future use of AI for chart analysis, the perception/reasoning split is the technical barrier being addressed here.
[Fundamentals]
Fundamentals: What arXiv Is — and Why AI Research Moves Before Peer Review

arXiv (pronounced "archive") is a free, open-access repository where researchers upload papers before they've been formally peer-reviewed or published in academic journals. It was founded in 1991 for physics and math; by 2026 it's the primary publishing venue for AI and machine learning research. When a researcher has a result they want to share, they post a "preprint" to arXiv the same day — often before anyone outside their lab has reviewed it.

This creates a structural anomaly: the most important AI research becomes public before it has been independently verified. The HyperP paper (Issue #7, claiming 3.38x compute efficiency) appeared on arXiv before replication attempts began. The PRCO paper above is the same. Peer review — the process where independent experts check the methodology and results before publication — typically takes 6–18 months in computer science. arXiv compresses that to zero.

For readers of this briefing, the practical implication is that arXiv papers should be treated on a sliding confidence scale: single-source arXiv claims are marked [Unconfirmed] until independently replicated. Papers from well-known labs (DeepMind, Microsoft Research, Shanghai AI Lab) with accompanying code releases replicate faster. Papers from unknown authors without code should be weighted lightly until corroborated.

The fast publication cycle is a feature, not a bug, for the AI industry: labs can stake out priority on an idea immediately, and practitioners can start building before formal publication. The PRCO framework above will be tested by the community within weeks. If it holds, it becomes real. If it doesn't, it fades. The community self-corrects faster than formal journals — just with more noise in the interim.

Why this matters for the AI landscape: The speed of arXiv is one of the main reasons AI capability is advancing faster than any other scientific field in history. It's also why AI benchmarks and claims are frequently overstated in media coverage — early arXiv results, reported breathlessly before replication, become "the new GPT-4" headlines that age poorly. When evaluating AI research stories, the first question is always: "Has this been replicated, and was there code?" If no to both, temper expectations.
